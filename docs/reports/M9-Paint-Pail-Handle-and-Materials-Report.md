# M9 — Paint-Pail Handle & Realistic Materials

**Milestone:** M9 (appearance)
**Scope:** Turn the bucket into a believable paint pail: add a procedurally generated metal
bail (handle) the rope hangs from, make the bucket an opaque galvanised-metal pail, and
finish the realistic liquid look.

All geometry and motion remain 100% hand-written (no Unity physics engine, no ready-made
asset packages). The GPU is used for rendering only.

---

## 1. Procedural bail handle

`BucketHandle.cs` is a new `[ExecuteAlways]` component that builds the curved metal handle
entirely in code — a thin tube swept along a half-ellipse over the bucket mouth:
- the arc ends at the two rim edges (±top radius) and peaks at an apex above the mouth,
- the tube is generated by sweeping a small ring along the arc (a proper round wire),
- it lives as a **child of the bucket**, so it tilts with it while swinging,
- its size stays in sync with the bucket automatically (rebuilt whenever the bucket rebuilds).

It is created/wired by a one-click editor command **Tools ▸ Liquid ▸ Bucket - Add Handle**,
which also gives it an opaque, double-sided dark-metal material (`BucketHandle.mat`).

Tunables on the `BucketHandle` component: `Rise` (arc height), `Thickness` (wire radius),
`Arc Segments`, `Tube Segments`.

## 2. Rope hangs from the bail apex

The rope now attaches to the **apex of the handle** (not the rim centre), like a real pail.
A real transform (`RopeApex`), a child of the bucket at the bail apex, is wired to the rope's
attach point; being a child it tilts with the bucket automatically. `Rope.cs` also computes the
apex live from the bail as a fallback.

**First-frame fix (execution order).** The tilt (`BucketTilt`) and the rope (`Rope`) both run in
`LateUpdate`. With the default, undefined order the rope could run *before* the tilt on the first
frame and draw to the apex on the not-yet-tilted bucket — a one-frame "rope off the arc" glitch.
`Rope` is now marked `[DefaultExecutionOrder(100)]` so it always runs *after* the tilt and reads
the bucket's final rotation. The rope stays glued to the bail apex from the very first frame.

## 3. Opaque galvanised-metal pail

The bucket wall material was changed from semi-transparent to an **opaque galvanised-metal**
look (light grey, metallic, double-sided so the inner walls still show through the open top).
This reads as a real metal pail; the liquid becomes visible from the open top and as it pours
out and paints the canvas — as with a real bucket.

## 4. Realistic liquid (finished)

- **Particle overlap:** `SphFluid.particleVisualScale` raised so neighbouring particles overlap
  and read as one connected body of liquid instead of separate balls.
- **Wet, glossy shading:** the hand-written `Custom/LiquidPaint` shader (Lambert + Blinn-Phong
  sheen + Fresnel rim) driven with a tight specular (`_Smoothness`) and a brighter specular
  colour for a clear wet highlight.
- **Larger bucket:** the procedural bucket was enlarged (top 0.70 / bottom 0.53 / height 1.50)
  so it holds more particles; the SPH container and auto-fill follow the size automatically.
