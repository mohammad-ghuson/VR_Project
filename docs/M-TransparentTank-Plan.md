# Transparent Liquid Tank (Supervisor request #3)

> Status: **PLAN — awaiting approval. No code yet.**
> Goal: a transparent rectangular box (cuboid tank) on the side, filled with SPH liquid,
> so the fluid motion is clearly visible. Built to be **movable/shakeable later** so the
> liquid sloshes (the user chose option B).

---

## 1. Idea & approach

A standalone transparent box, separate from the bucket, with its own SPH liquid. We reuse
our existing `SphFluid` solver but add a **BOX boundary** mode (today it only supports a
cylinder). A box boundary is actually simpler than the cylinder: clamp the particle inside
the box's half-extents on each axis and bounce off the face it crossed.

To support sloshing later, the box boundary is **oriented** and follows the tank's transform
(position + rotation), exactly like the bucket's moving cylinder — so moving/rotating the
tank shoves the fluid (wall-relative bounce we already have).

Constraints unchanged: pure C# math, no physics engine, no ready-made packages; the box mesh
is generated by us; transparency is standard URP rendering.

---

## 2. Steps (small, testable)

### Step T1 — Transparent box object
- Generate a cuboid mesh ourselves (faces + edges) and a **transparent** material so we see
  inside.
- A `PaintTank` GameObject placed to the side of the scene.
- Editor menu to create it at a chosen size.
- **Test:** a see-through box appears on the side; nothing else changes.

### Step T2 — Box boundary in SPH + fill the tank
- Add to `SphFluid`: a `containerShape` choice (Cylinder / Box) and `boxHalfExtents`.
- Implement box boundary resolution (clamp within the oriented box, bounce off faces,
  reflecting velocity relative to the wall — same mechanism as the bucket).
- A second `SphFluid` instance on the tank, shape = Box, filling it with particles.
- **Test:** liquid fills the box, settles flat, clearly visible through the walls.

### Step T3 — Make the tank shakeable (option B)
- The box container follows the tank transform; moving/rotating the tank sloshes the liquid.
- **Test:** drag/rotate the tank in the Scene view → the liquid sloshes and waves, then
  settles. (Auto-shake or hooking to motion can come later.)

---

## 3. Design notes
- The tank is independent of the bucket. If two SPH instances are too heavy, we can disable
  the bucket's SPH while demoing the tank.
- Box is **open-topped** (like the bucket) so we never trap pressure; fill level kept below
  the rim.
- The existing **cylinder** path stays untouched — box is a separate branch.

## 4. Out of scope (later)
- Auto-shake / linking the tank to a motion script (easy follow-up).
- Canvas drawing (M4) and UI (M5) — next milestones per the supervisor.

## 5. Risks
- Performance with two fluids → keep particle counts modest; can disable one.
- Transparency sorting in URP → use a proper transparent material/queue.
